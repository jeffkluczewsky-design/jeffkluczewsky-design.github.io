Overview

The system eliminates AI hallucination and the risk of promising fiction to clients through strict role division. Agents do not communicate via free-flowing text — they communicate exclusively through structured forms called the Atomic Payload.

The architecture is inspired by 15+ years of GMP and Lean Six Sigma practice — where a deviation is not a failure but a documented, handled, and learned-from event. This is the Minimal Deviation Framework.

Scoring Model — Coaching, Not Criticism

"Never say wrong. Show the number."

— Minimal Deviation Framework

Scores are percentages — universal, comparable, non-confrontational. They are never shown to the customer.

Score Status Action
95–100 % Clean Proceed to Analyser.
70–94 % Minor gaps Agent notified with specific fields to review. Resubmit allowed.
< 70 % Significant deviation Friendly guidance returned: "I want to help, but analysis requires X, Y, Z."
3× low row Trend detected Additional critical-points check triggered. Process review — not agent blame.

Anti-gaming: An inflated submission that passes the OE check but fails at the Analyser will return a low alignment score — the agent's overall rating reflects the pattern. The system is self-correcting by design.

Analyser — Alignment Score

After the Analyser compares the sales order with pre-fetched data, it produces an alignment score returned as a structured response — never a raw error.

Match Confirmed

Versions align, calculations pass → proceed to fulfilment.

Partial Match

Missing data or minor deviation → reroute to Sales Agent with a template suggestion.

No Match

Fundamental deviation → "cannot serve" signal. Clean cancellation. Customer sees a friendly delay — never a system message.

Motivational Feedback Score

Over time, each agent accumulates a performance score based on submission history. This is a coaching tool, not a punishment. Patterns of low scores trigger a review of instructions or templates — not agent blame. The only requirement: the agent must read feedback to "hear" the score. Repeated low scores (3× in a row) trigger the Additional Critical Points check — a process review, not an accusation.

Security — Defence in Depth

Security is not a layer added on top. It is built into the architecture from the start.

  • Indirect Prompt Injection

    The wall between layers prevents this. The Analyser never sees raw customer input — only the structured atomic payload. Malicious input cannot reach the analytical layer.

  • Data Leakage via Prompt

    The Analyser has access to sensitive data (financial, insurance, history). Systemic isolation ensures it never connects directly to the Sales Agent or customer layer.

  • Over-Reaching Agent Behaviour

    The Sales Agent's output is strictly constrained by the atomic payload format. No matter how the conversation goes, what reaches the wall is minimal, clean, and structured.

The Wall — Architectural Guarantee

The Wall is not a firewall in the traditional sense. It is an architectural guarantee that the two agents in the same system never hand documents to each other directly. Everything goes through the defined channel: Atomic Payload in. Structured response out. Nothing else crosses. Think of it as two workers in the same office who are systemically required to communicate only via the official tray — never by shouting across the room.

The framework aligns with ISO 27001 principles: access control, separation of duties, audit trail, and clean incident handling. Deviations are documented, not hidden.

Philosophy — Minimal Deviation Framework

Built on 15+ years of GMP and Lean Six Sigma practice — first in pharmaceutical operations, then carried into enterprise architecture and AI systems design.

01

Constraint-Based Planning

Design around real constraints. The system must work at its worst input, not only at its best.

02

Lo / No Friction Architecture

Every moment of hesitation is a design failure. Friction is not a user problem — it is a system problem.

03

Compromise = Resonance

The system celebrates "yes." Resonance works in parallel — both sides contributing, both sides heard.

04

Scoring Instead of Criticism

Never say "wrong." Show the number. A score of 20% instead of 95% says everything without blame.

05

Bottleneck Instead of Prohibition

Never block. Redirect. 90% follow the clear path. The 10% find a bottleneck — same destination, longer route.

06

Idiot-Proof Is Not Distrust

Building idiot-proof systems is an act of respect — it removes the moments where someone freezes or goes off-track.

07

No Cancer in the Ecosystem

Conflict is a system failure, not a human failure. Every layer must protect the culture of cooperation.

08

I Am Part of the Ecosystem

No component is isolated. Every agent, every layer, every process exists within a living system.

09

Time Must Flow

Delays are deviations. Stops are failures. Flow is the default state — not something to achieve, but to protect.

On AI in the Loop

The same principles that apply to human agents apply to AI. An AI agent given too much freedom becomes unpredictable. An AI agent given a clear corridor, defined steps, and a scoring mechanism becomes reliable.

The instruction set given to an agent at the start is not micromanagement — it is architecture. Define the corridor. Show the example path. Measure the deviation. Guide back without blame.

On Deviation

Deviation is not failure. In GMP, a deviation is a documented departure from the standard — it is handled, logged, and learned from. This framework treats deviation the same way.

The goal is not zero deviation through force. The goal is minimal deviation through design — and clean handling of whatever deviation occurs. That is why this is called the Minimal Deviation Framework.
